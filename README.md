# IFRS 9 ECL Calculator

A full-stack demo that classifies a loan portfolio into IFRS 9 Stage 1 / 2 / 3
and calculates Expected Credit Loss (ECL) at loan and portfolio level.

- **Backend**: Python + FastAPI for the staging logic and ECL engine
- **Frontend**: React + Vite + TypeScript dashboard (summary metrics,
  staging/ECL charts, filterable loan table, CSV upload)

> **Disclaimer**: This is a simplified, illustrative implementation built for
> portfolio purposes. It is **not** a production-grade IFRS 9 model and
> should not be used for regulatory reporting. All data is synthetic.

---

## Methodology

Each loan has the following inputs: `product_type` (the portfolio segment
the loan belongs to), `exposure_at_default` (EAD), `lgd`, `pd_12m` (current
12-month PD), `pd_origination` (PD at origination), `days_past_due`,
`remaining_term_months`, and `eir` (effective interest rate, used for
discounting).

In practise ECL is calculated separately for each portfolio (mortgages,
personal loans, SME etc), since each has its own PD/LGD models and
parameter ranges. `product_type` is a free text field for whatever label
your portfolio uses - the bundled sample is a single retail personal loan
portfolio, generated by `app/data_generator.py`.

### Staging

| Stage | Criteria |
|---|---|
| **Stage 3** | `days_past_due >= 90` (credit-impaired / default) |
| **Stage 2** | `days_past_due >= 30` OR `pd_12m / pd_origination >= 2.0` (significant increase in credit risk) |
| **Stage 1** | Otherwise |

### ECL Calculation

- **Lifetime PD** is approximated by compounding the 12-month PD over the
  remaining term: `pd_lifetime = 1 - (1 - pd_12m) ^ (remaining_term_months / 12)`, capped at 1.0.
- **Stage 1** → 12-month ECL = `pd_12m * lgd * exposure_at_default`
- **Stage 2 / 3** → Lifetime ECL = `pd_lifetime * lgd * exposure_at_default`

### Discounting to Present Value

IFRS 9 requires ECL to be discounted to present value at the loan's
effective interest rate (EIR), for both 12-month and lifetime ECL. This
implementation doesn't model a full cash shortfall schedule, so as a
simplification it assumes the loss crystallises at a single point within
the relevant ECL horizon. Two timing assumptions are supported, selectable
via the `discount_method` parameter (or the dropdown on the dashboard):

- **`midpoint`** (default): the loss crystallises half way through the
  horizon (0.5 years for stage 1, mid-point of the remaining term for
  stage 2/3)
- **`end_of_horizon`**: the loss crystallises at the end of the horizon
  (1 year for stage 1, the full remaining term for stage 2/3). This is
  more conservative, giving a higher PV for the same undiscounted ECL.

`ECL = (PD x LGD x EAD) / (1 + eir) ^ years_to_loss`

Both the discounted and undiscounted ECL figures are shown throughout the
app, so the impact of discounting is visible at a glance. Portfolio-level
results aggregate total exposure, total discounted and undiscounted ECL,
the ECL coverage ratio, and counts/exposure/ECL broken down by stage.

## Running Locally

### Backend

```bash
cd backend
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload
```

The API runs at `http://localhost:8000`.

Run the test suite:

```bash
pytest
```

### Frontend

```bash
cd frontend
npm install
npm run dev
```

The dashboard runs at `http://localhost:5173` and expects the backend API at
`http://localhost:8000`.

## API Endpoints

| Method | Path | Description |
|---|---|---|
| `GET` | `/api/portfolio` | Returns the bundled synthetic sample portfolio, processed (staging + ECL) |
| `POST` | `/api/portfolio/upload` | Accepts a CSV upload of a custom loan portfolio, returns processed results |
| `GET` | `/api/portfolio/sample-csv` | Downloads the sample portfolio CSV as a template for custom uploads |

Both `/api/portfolio` and `/api/portfolio/upload` accept an optional
`discount_method` query parameter (`midpoint` or `end_of_horizon`,
default `midpoint`) to control the discounting timing assumption described
above.

### CSV Format

Custom portfolio CSVs must include the following columns:

```
loan_id, product_type, exposure_at_default, lgd, pd_12m, pd_origination, days_past_due, remaining_term_months, eir
```

`product_type` can be any text label for your portfolio segment (e.g.
`mortgage`, `sme_term_loans`). Since ECL is normally calculated separately
per portfolio, a single CSV upload would typically contain one
`product_type` value, though the dashboard will happily filter and display
a mix if you give it one.
